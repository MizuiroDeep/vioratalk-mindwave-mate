## English

### About This Project

VioraTalk is a personal learning project where I'm developing an AI voice assistant with Claude AI's help. As someone still learning Python, I deeply appreciate any form of contribution that helps make this project better.

### Current Project Status

This project is under active development. While I'm not able to accept all pull requests due to my limited experience, there are many ways you can help:

### Ways to Contribute

#### 🌟 Star the Repository
If you find this project interesting or useful, please give it a star! It helps others discover the project and motivates me to continue development.

#### 💬 Join Discussions
Have ideas, questions, or feedback? Please start or join a discussion in the [Discussions](https://github.com/MizuiroDeep/vioratalk-mindwave-mate/discussions) tab. I'm particularly interested in:
- Feature ideas and use cases
- User experience feedback
- Technical suggestions (though I might need time to understand them!)
- General encouragement and support

#### 🐛 Report Bugs
Found something that doesn't work? Please report it! When reporting bugs:
1. Check if the issue already exists
2. Include the error message and error code (if any)
3. Describe what you were trying to do
4. List your environment (Windows version, Python version)
5. Provide steps to reproduce the issue

I'll do my best to fix bugs, though it might take some time as I learn.

#### 📝 Improve Documentation
If you notice any of the following, please let me know:
- Typos or grammatical errors
- Unclear explanations
- Missing information
- Better ways to explain concepts

#### 💡 Suggest Features
Have an idea for VioraTalk? I'd love to hear it! Please:
- Explain the feature and why it would be useful
- Provide examples of how it might work
- Be patient - complex features might take time to implement

### Pull Requests

While I appreciate the effort people put into pull requests, I'm currently not accepting most PRs. This is because:
- I'm still learning Python and need to understand all code in the project
- Working through problems myself is part of my learning process
- I want to maintain consistency in the codebase while I'm learning

However, I will consider PRs for:
- Typo fixes in documentation
- Critical bug fixes with clear explanations
- Translation improvements

If you want to submit a PR:
1. Please open an issue first to discuss the change
2. Keep changes small and focused
3. Include clear explanations of what and why
4. Be patient with my review process

### Code Style

If you do submit code:
- Follow PEP 8 style guidelines
- Include docstrings for functions
- Add type hints where possible
- Write tests for new functionality
- Keep it simple - remember I'm still learning!

### Questions and Support

If you have questions about the project:
- Check the documentation first
- Look through existing discussions
- Open a new discussion if you can't find an answer
- Be patient - I'll respond when I can

### Recognition

All contributors will be acknowledged in the project documentation. Your support, in any form, is greatly appreciated!

## Japanese

### このプロジェクトについて

VioraTalkは、私がClaude AIの助けを借りながら開発している音声アシスタントの個人学習プロジェクトです。Pythonを勉強中の身として、プロジェクトをより良くするあらゆる形の貢献に深く感謝しています。

### 現在のプロジェクト状況

このプロジェクトは活発に開発中です。経験が限られているため、すべてのプルリクエストを受け入れることはできませんが、様々な方法で協力していただけます：

### 貢献の方法

#### 🌟 リポジトリにスターを付ける
プロジェクトが面白い、または役立つと思ったら、ぜひスターを付けてください！他の人がプロジェクトを見つけやすくなり、開発を続けるモチベーションにもなります。

#### 💬 ディスカッションに参加
アイデア、質問、フィードバックがありますか？[Discussions](https://github.com/MizuiroDeep/vioratalk-mindwave-mate/discussions)タブでディスカッションを始めたり、参加したりしてください。特に以下のような内容を歓迎します：
- 機能のアイデアや使用例
- ユーザー体験のフィードバック
- 技術的な提案（理解するのに時間がかかるかもしれませんが！）
- 一般的な励ましやサポート

#### 🐛 バグを報告
動作しないものを見つけましたか？ぜひ報告してください！バグ報告の際は：
1. 既存のissueがないか確認
2. エラーメッセージとエラーコード（ある場合）を含める
3. 何をしようとしていたか説明
4. 環境を記載（Windowsバージョン、Pythonバージョン）
5. 再現手順を提供

学習しながらですが、できる限りバグを修正します。

#### 📝 ドキュメントの改善
以下のような点に気づいたら、お知らせください：
- 誤字や文法の間違い
- 不明確な説明
- 不足している情報
- より良い説明方法

#### 💡 機能の提案
VioraTalkのアイデアがありますか？ぜひ聞かせてください！以下をお願いします：
- 機能とその有用性を説明
- 動作例を提供
- 複雑な機能の実装には時間がかかることをご理解ください

### プルリクエスト

プルリクエストに費やされる努力に感謝しますが、現在ほとんどのPRを受け入れていません。理由は：
- Pythonを学習中で、プロジェクト内のすべてのコードを理解する必要がある
- 問題を自分で解決することが学習プロセスの一部
- 学習中はコードベースの一貫性を保ちたい

ただし、以下のPRは検討します：
- ドキュメントの誤字修正
- 明確な説明付きの重要なバグ修正
- 翻訳の改善

PRを送信する場合：
1. まずissueを開いて変更について議論してください
2. 変更は小さく、焦点を絞って
3. 何を、なぜ変更するのか明確に説明
4. レビュープロセスに時間がかかることをご了承ください

### コードスタイル

コードを提出する場合：
- PEP 8スタイルガイドラインに従う
- 関数にdocstringを含める
- 可能な限り型ヒントを追加
- 新機能にはテストを書く
- シンプルに保つ - 私はまだ学習中です！

### 質問とサポート

プロジェクトについて質問がある場合：
- まずドキュメントを確認
- 既存のディスカッションを確認
- 答えが見つからない場合は新しいディスカッションを開く
- 返信に時間がかかることをご了承ください

### 謝辞

すべての貢献者はプロジェクトドキュメントで認識されます。どのような形であれ、あなたのサポートに深く感謝しています！
